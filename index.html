COMANDOS BASICOS GIT

git --version //muestra la version de git instalada
git init //inicializa un repositorio de git, generalmente la carpeta actual
git status // muestra el estado de las modificaciones en los archivos
git status -s //Muestra resumido en colores
git add "nombre de archivo" //sin comillas, agrega "nombre de archivo" al stage
git add . //agrega todos los archivos de la carpeta inicializada al stage
git reset "nombre de archivo" //sin comillas, quita el archivo del stage
git reset . //quita todos los archivos del stage
git restore --staged "nombre del archivo" //sin comillas
git commit -m "mensaje del commit con sentido de lo q se ha hecho"
git commit --amend //Actualiza el mensaje del ultimo commit
git checkout -- . //regresa al estado del ultimo commit realizado, aunq se hayan grabado cambios en el archivo
git diff --stage
git log //revisar el historial del repositorio
git log --oneline
git branch //muestra las ramas existentes y en cual se esta trabajando

Notas

Git restore se puede usar para restaurar archivos en el árbol de trabajo o área de preparación a un estado anterior. Git reset se puede usar para volver a un commit concreto, descartando todos los commits realizados después de ese. git restore restablece el archivo a su contenido anterior, perdiendo los cambios que realizaras. git reset en su lugar, lo quita de la zona de preparación del próximo commit

Git restore apareció por primera vez en agosto de 2019, en Git 2.23. Git reset ha estado en Git desde antes de 2005.

Antes de utilizar git reset, es importante tener en cuenta el tipo de cambios que piensas hacer.
